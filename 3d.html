Overview

A living garden grown entirely from mathematics.

This project is a generative 3D garden built with p5.js WEBGL. Every tree, flower, butterfly, and stone path is positioned and animated dynamically using mathematical functions — no assets, no prefabs. Parametric equations govern petal placement, recursive fractals shape branching trees, and sine waves animate butterfly wings. Users can rotate and zoom to explore the garden from any angle, revealing the geometry underneath the organic forms.

Most 3D environments are built from pre-made assets — models, textures, imported geometry. This project takes the opposite approach: every element is described mathematically and rendered from scratch at runtime. The garden exists as a set of equations rather than files.

The challenge was making mathematical structures feel organic — preventing the rigidity of perfect symmetry from breaking the illusion of a living space. Controlled randomness, the Golden Angle, and fractal recursion each play a role in making equations look like nature.

Mathematics as gardener — equations that grow rather than describe.

Garden Elements

Six elements,
one world

E.01
Trees

Recursive fractal branching with scaled branch lengths. Each generation shrinks by 0.75× creating naturalistic taper from trunk to twig.

E.02
Flowers

Petals placed via parametric polar equations using the Golden Angle (137.5°) to prevent overlap and create natural Fibonacci-pattern blooms.

E.03
Bushes

Spherical clusters of geometry positioned using randomised polar coordinates within a bounded radius, creating natural clumping.

E.04
Stone Path

Elliptical stone placement with added noise — x = r cos(?) + rand(-40,40) — giving the path a hand-laid, winding quality.

E.05
Butterflies

Sine-based wing flap animation with bilateral symmetry. Position follows a slow parametric arc through the scene space.

E.06
Grass Plane

A ground plane with distributed grass blades, each slightly rotated using noise functions to break uniform alignment.

Visual Output

Interaction

Users can rotate the scene freely using mouse drag, and zoom with scroll. The 3D WEBGL camera allows full orbital navigation around the garden, revealing depth and structure invisible from any single angle.

Generative variation

Each time the sketch is run, randomness seeds produce a unique garden. Tree heights, flower counts, stone placements, and butterfly paths all vary — the equations stay fixed but the output never repeats exactly.

04
Under the hood

The math
behind it

M.01
Tree branching
x = x + L·cos(?)   y = y - L·sin(?)   L? = 0.75·L??1

Recursive growth using angle-driven offsets, with each generation's branch length scaled by 0.75 of the parent — producing the natural taper from trunk to twig tip.

M.02
Flower petals — Golden Angle
x = r·cos(i × 137.5°)   y = r·sin(i × 137.5°)

The Golden Angle (˜137.5°) is irrational with respect to a full rotation, meaning no two petals ever land in the same angular slot. The result is the Fibonacci spiral pattern found in real sunflowers and daisies.

M.03
Butterfly wing animation
flapAngle = sin(t) × 10°

Wings are mirrored geometry rotated by a sine function of time — producing a smooth, periodic flap. The bilateral symmetry is maintained by reflecting the same geometry across the x-axis.

M.04
Stone path — elliptical with noise
x = r·cos(?) + rand(-40,40)   y = r·sin(?) + rand(-40,40)

An ellipse provides the base path geometry, but added uniform random noise displaces each stone slightly — mimicking the imprecision of hand-laid garden paths without losing the overall winding structure.

M.05
Clouds & leaves — fractal distribution
Julia & Mandelbrot iteration sets

Julia and Mandelbrot fractal iteration determines cluster density and shape for cloud formations and leaf distributions — creating the irregular, self-similar patterns found in real foliage and cumulus clouds.

Built with p5.js WEBGL

WEBGL mode in p5.js gives access to a proper 3D coordinate system and perspective camera. Every element is drawn using 3D primitives and custom geometry — no 2D canvas tricks.

Balancing visual richness with WEBGL performance was the central tension throughout. Each recursive tree branch, each fractal leaf cluster, each animated butterfly adds draw calls — pushing against the limits of what runs smoothly in a browser tab.

The other core challenge was controlled naturalness — preventing mathematical regularity from reading as artificial. The Golden Angle solves this for flowers, noise functions solve it for paths, and asymmetric recursion solves it for trees. Each element needed its own strategy for hiding the grid underneath.

The hardest part wasn't the equations — it was making equations feel like they weren't there.
