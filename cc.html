Overview

Making the cosmos legible — one machine learning model at a time.

Cosmic Conversion is a full-stack machine learning project that transforms open-source scientific data into an interactive platform with three modules: Stars (spectral classification), Waves (gravitational wave analysis), and Worlds (exoplanet habitability). Originally prototyped with Plotly Dash, the platform was rebuilt using a Python FastAPI backend and React.js frontend, enabling scalable and modular integration of multiple ML models.

Astronomical datasets are often large, inconsistent, and difficult to interpret — limiting their accessibility to the broader scientific community and enthusiasts alike. The challenge was to build a system that could clean, normalise, and merge multiple open-source datasets, train machine learning models across classification and regression tasks, and present results in an intuitive, interactive platform.

Integrating backend ML pipelines with a React.js frontend while maintaining high performance and visual clarity in data presentation posed a significant engineering and design challenge — requiring iterative testing across both the analytical and experiential layers.

The universe generates endless data. The question is whether we can make it speak.

03
Project Architecture

Three modules,
one platform

Stars Module

Predicts stellar properties (temperature, radius, spectral type) from observed light using RandomForestRegressor and Linear Regression. Inputs include color index, magnitude, and luminosity — outputs include HR diagram placement.

Waves Module

Detects gravitational-wave signals and estimates distances from cosmic events. Uses a Random Forest Classifier for signal/noise classification and a Regressor for distance estimation in megaparsecs.

Worlds Module

Predicts exoplanet habitability from planetary parameters using Logistic Regression, K-Means clustering, and Gradient Boosting. Outputs a habitability score (0—1) and classification.

Challenges

Constraints &
honest limits

C.01
Dataset Complexity

Large and inconsistent datasets across three domains required extensive cleaning, normalisation, and validation before training could begin.

C.02
Backend—Frontend Integration

Connecting FastAPI ML pipelines with the React frontend introduced performance and communication challenges requiring careful API design.

C.03
Analytical Accuracy vs Clarity

Balancing ML model accuracy with intuitive, clear visualisation demanded iterative testing across both the analytical and UX layers.

C.04
Modular Architecture

Designing three independent ML modules to share a single platform without coupling required disciplined modular development and testing.
